# Hive proposals inbox

**Purpose:** propose upgrades to **lk-bloom/hive itself** (Master Build Pattern, pool pack, doctrine, distill, orchestration bible, roadmap).  
**Not** a product WORKSTREAMS. Product ship ideas stay in product pool PROPOSALS.

**Steward:** Admin (human) or designated Bloom Steward accepts / parks / rejects.

---

## How to file

1. Copy [`PROPOSAL.template.md`](./PROPOSAL.template.md) → `YYYY-MM-DD-<slug>.md` in this folder.
2. Link evidence (CLOSEOUT digests, audits, instance pain).
3. Suggest bump class: patch · minor · major (see MBP version rules).
4. Leave status `open` until Steward acts.

---

## Status values

`open` · `acked` · `parked` · `accepted` · `rejected`

When **accepted**: Steward applies the change under `hive/`, bumps doctrine version if needed, notes INSTANCES upgrade guidance, and links the proposal from the changelog.
